v1.1.0

feat: Rewrite the application backend in Rust with an Axum/SQLx service that preserves the existing API and single-container deployment contract.
feat: Port authentication, OIDC, security headers, gzip, admin/debug tooling, bootstrap metadata, preferences, site settings, and live state events to the Rust runtime.
feat: Port folder browsing, document metadata, version history, locks, checkouts, archive/restore, favorites, sharing, retention, uploads, downloads, and export workflows to Rust.
feat: Add Rust-native resumable transfer sessions, transfer cleanup, folder export jobs, ZIP64 archive support, and local/S3/R2 object storage handling.
docs: Document the Rust rewrite and update Docker release defaults for v1.1.0.

v1.0.1

fix: Harden SQLite event streaming under concurrent use by moving state-event database reads off the ASGI event loop.
fix: Wake idle SSE clients from committed state-change notifications instead of per-client polling.
fix: Align SQLite busy timeout and SQLAlchemy pool configuration for the single-container deployment profile.
fix: Keep file extensions out of the default rename selection unless the user explicitly edits them.
fix: Show Remove from Favorites when a selected item is opened from the Favorites context menu.
fix: Prevent archived files from being renamed in the UI, inline rename flow, and rename API.
